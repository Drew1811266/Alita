# Alita 工程系统与首选项设计

## 目标

为 Alita 增加真正的工程文件工作流和全局首选项界面，让软件启动后更像一款专业桌面生产工具，而不是直接进入一个临时工作台。

## 已确认决策

- 软件启动后首先显示“工程主页”。
- 用户必须新建或打开工程后，才能进入工作台。
- 第一版工程格式采用单个 `.alita` JSON 文件。
- 大型源文件不嵌入 `.alita` 文件；工程文件只保存文件路径引用和附件元数据。
- 首选项可以从工程主页和工作台顶部栏进入。
- 模型管理支持手动添加 `.gguf` 模型，也支持扫描用户配置的模型目录。
- 工具/节点管理显示已安装工具的 manifest 信息，并允许用户启用或禁用 Agent 可使用的工具。

## 用户体验

### 工程主页

软件启动后的第一个界面是工程主页。它包含：

- 软件标题：`Alita`
- 主要操作：`新建工程`
- 次要操作：`打开工程`
- 最近工程列表
- 右上角 `首选项` 按钮

用户在没有激活工程之前，不能进入聊天和节点工作台。这样可以确保聊天记录、节点图、文件引用、模型选择和执行历史都归属于明确的工程。

### 工作台

打开工程后，现有工作台仍然是主要生产界面：

- 顶部栏显示工程名、保存状态、`保存`、`另存为` 和 `首选项`
- 左侧聊天区域占 40%
- 右侧节点画布占 60%

工作台会从当前 `.alita` 文件加载初始聊天记录和节点图。新建工程时，默认使用当前开发版的初始提示消息，并显示空节点画布。

### 首选项

首选项是全局配置，不属于某一个工程。用户可以在打开工程之前或打开工程之后进入首选项。

首选项第一版包含以下分区：

- `模型`
- `工具节点`
- `Agent`
- `安全`

第一版只需要让 `模型` 和 `工具节点` 分区具备实际功能。`Agent` 和 `安全` 可以作为后续规划入口出现，但不暴露未完成的可操作控件。

## 工程文件格式

`.alita` 文件是 UTF-8 JSON 文档。文件扩展名面向用户，内部格式第一版保持简单、可检查、可调试。

```json
{
  "schemaVersion": 1,
  "projectId": "project-uuid",
  "name": "文档整理测试",
  "createdAt": "2026-05-09T12:00:00.000Z",
  "updatedAt": "2026-05-09T12:10:00.000Z",
  "messages": [],
  "graph": null,
  "attachments": [],
  "modelRef": null,
  "toolSnapshot": [],
  "runHistory": []
}
```

### 必需字段

- `schemaVersion`：工程文件结构版本。第一版为 `1`。
- `projectId`：创建工程时生成的稳定 UUID。
- `name`：显示在工程主页和工作台顶部栏中的工程名称。
- `createdAt`：ISO 时间戳。
- `updatedAt`：每次保存时更新的 ISO 时间戳。
- `messages`：聊天消息，沿用当前前端 `ChatMessage` 结构。
- `graph`：节点图，沿用当前前端 `NodeGraph` 结构；没有节点图时为 `null`。
- `attachments`：已知附件元数据和原始文件路径。
- `modelRef`：选中的模型 id；未选择时为 `null`。
- `toolSnapshot`：保存时的工具启用状态和版本快照。
- `runHistory`：用于后续检查的最小执行记录。

### 文件引用策略

第一版只保存用户文件路径，不复制或嵌入文件本体。如果打开工程时发现某个引用路径已经不存在，工程仍然可以打开，但需要提示用户重新定位该文件。

这样可以避免 `.alita` 文件体积过大，同时保留足够的元数据，让 Agent 和 UI 能解释缺失了什么。

## 首选项数据

首选项独立于工程文件保存，因为模型路径和工具启用状态属于当前机器的全局配置。

第一版推荐保存位置：

- Tauri 应用配置目录
- 文件名：`preferences.json`

初始结构：

```json
{
  "schemaVersion": 1,
  "modelDirectories": [],
  "models": [],
  "defaultModelId": null,
  "toolEnablement": {}
}
```

### 模型注册表

每个模型条目记录本地元数据：

```json
{
  "modelId": "model-uuid",
  "name": "Qwen3 8B",
  "path": "D:\\Models\\qwen3-8b.gguf",
  "source": "manual",
  "runtime": "llama_cpp",
  "fileExists": true,
  "createdAt": "2026-05-09T12:00:00.000Z",
  "updatedAt": "2026-05-09T12:00:00.000Z"
}
```

手动添加会记录一个 `.gguf` 文件。目录扫描会在用户配置的目录中搜索 `.gguf` 文件，并以 `source: "scan"` 添加发现的模型。如果同一路径已经注册，扫描器只更新元数据，不重复添加。

### 工具启用状态

工具启用状态以工具 manifest 中的 `tool_id` 为键：

```json
{
  "document.process": true,
  "image.process": false
}
```

缺失条目的默认策略：

- 内置且稳定的工具默认启用。
- 占位类工具默认禁用。

Agent 规划任务时只能考虑已启用的工具。

## Rust/Tauri 责任

Rust 负责可信桌面操作：

- 创建新的 `.alita` 工程文件
- 打开并校验已有 `.alita` 工程文件
- 保存当前工程状态
- 将当前工程另存为新的 `.alita` 路径
- 维护最近工程路径
- 读取和写入全局首选项
- 扫描模型目录中的 `.gguf` 文件
- 暴露已安装工具 manifest 的摘要信息
- 更新工具启用状态

前端不直接通过浏览器 API 写入工程文件。前端应调用 Tauri command，并接收类型化 JSON 响应。

## 前端责任

React 负责用户流程和本地 UI 状态：

- 没有激活工程时显示工程主页
- 从工程主页或工作台打开首选项
- 根据已加载的工程数据初始化工作台状态
- 当消息、节点图、选中模型或附件元数据变化时，将工程标记为未保存
- 从顶部栏触发保存命令
- 为缺失的引用文件显示警告
- 在首选项中渲染模型列表和工具列表

现有聊天组件和节点画布组件应继续保持聚焦。工程主页和首选项功能应放在独立 feature 目录中，避免把 `App.tsx` 扩张成过大的控制器。

## 数据流

### 新建工程

1. 用户点击 `新建工程`。
2. 前端请求 Rust 创建新的工程文件路径。
3. Rust 写入 schema version 1 的 `.alita` 文件。
4. 前端切换到工作台，并初始化默认聊天消息和空节点图。
5. 工程顶部栏显示工程名称和已保存状态。

### 打开工程

1. 用户点击 `打开工程`。
2. Rust 读取并校验用户选择的 `.alita` 文件。
3. Rust 返回类型化工程数据和非致命警告。
4. 前端加载消息、节点图、附件、模型引用和工具快照。
5. 最近工程列表更新。

### 保存工程

1. 用户点击 `保存`，或未来的自动保存触发。
2. 前端将当前消息、节点图、附件、选中模型引用和工具快照发送给 Rust。
3. Rust 以原子写入方式更新 `.alita` 文件。
4. 前端清除未保存状态。

### 首选项

1. 用户打开 `首选项`。
2. 前端向 Rust 请求首选项、模型列表和工具摘要。
3. 模型变更写入全局首选项。
4. 工具启用状态变更写入全局首选项，并影响后续 Agent 规划。
5. 已打开工程保留保存时的 `toolSnapshot`，但 Agent 默认使用当前全局工具启用状态。未来如果需要，可以再增加工程级覆盖配置。

## 错误处理

- `.alita` JSON 无效：显示错误，并停留在工程主页。
- `schemaVersion` 不支持：显示错误，说明该文件来自不兼容版本。
- 引用文件缺失：允许打开工程，并显示警告。
- 保存失败：保留未保存状态，并显示失败的文件路径。
- 模型路径缺失：模型仍显示在列表中，但标记为文件缺失。
- 扫描目录不可访问：保留已有模型，并显示扫描警告。
- 工具 manifest 无效：该工具不进入 Agent 规划，并在首选项中标记为无效。

## 安全边界

- `.alita` 加载时必须先校验路径和 schema，再使用其中的数据。
- 打开工程不能执行任何脚本。
- 工具启用状态只表示 Agent 是否可以在规划中使用该工具，不额外授予文件系统或进程权限。
- 临时脚本节点仍然必须经过 AI 安全审查，并得到用户明确授权后才能执行。
- 模型目录扫描只读取 `.gguf` 文件的元数据和路径。

## 测试策略

### Rust

- 工程文件序列化与反序列化
- schema 校验失败
- 原子保存行为
- 最近工程列表更新
- 首选项读取与写入
- 模型目录扫描和重复路径处理
- 工具启用状态默认值

### 前端

- 没有激活工程时渲染工程主页
- 打开工程后切换到工作台
- 聊天或节点图变化后显示未保存状态
- 保存后清除未保存状态
- 可以从工程主页和工作台打开首选项
- 模型列表能显示手动添加和扫描发现的模型
- 工具列表能显示启用、禁用和无效状态

### 集成验证

- 新建工程，发送聊天任务，生成节点图，保存，重新打开，确认状态恢复。
- 在首选项中添加模型路径，重新打开首选项，确认模型持久化。
- 禁用一个工具，确认前端首选项中显示禁用状态。

## 第一版不包含

- 将源文档、音频、图片或视频嵌入 `.alita`
- 导出工程包，例如 `.alitapkg`
- 云同步
- 多人协作编辑
- 模型下载管理器
- 本地模型性能测试器
- 工程级工具启用状态覆盖
- 面向未来 schema 版本的自动迁移

## 实现备注

第一版实现要保持收敛。当前目标不是构建完整的素材资产管理系统，而是让桌面软件具备真实的工程文件工作方式，并建立后续 Agent 和工具系统可以依赖的持久化边界。
